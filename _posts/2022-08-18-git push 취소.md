- 잘못된 commit을 이미 원격저장소에 push한 경우 되돌리는 방법  
- 로컬의 커밋을 취소하고, 강제로 push 하는 방식이기 때문에 main브랜치에서는 사용하지 않는것이 좋다

## [local] commit 취소
```
// 가장최근의 commit 취소
$ git reset HEAD^

// 최초 commit 취소
git update-ref -d HEAD  
git rm --cached -r . 

// 특정 커밋으로 이동
$ git log
$ git log --oneline --graph

$ git reset [commit id]
```

## 저장소에 push
```
// 필요하다면 커밋
$ git commit -m "commit messages"

$ git push origin [branch name] -f
```
